## 문서 사용 규칙

- **UI ID 체계**: `UI-{컴포넌트}-{순번}` — 예: `UI-SIDE-01`, `UI-INSP-T1-02`
- 이 문서는 비전검사 대시보드(v1.1 추가) UI/UX 설계를 중심으로 기술한다.
- 모델 탐색 대시보드 UI는 기존 구현을 그대로 유지하며, 충돌이 발생하는 항목만 이 문서에서 명시한다.
- 모든 컬러 코드는 Streamlit 기본 테마 기준이며, dark/light 양쪽에서 동작해야 한다.

## B. 사이드바 (v1.1 변경)

### B.1 변경 내용

v1.0에서 사이드바는 데이터셋 경로, 디바이스 정보, 전처리/모델 설정 요약을 표시했다.
v1.1에서 사이드바는 **대시보드 전환 버튼만** 표시한다. 기존 정보 섹션은 제거된다.

> **주의**: 이 변경은 모델 탐색 대시보드에도 영향을 미친다. 기존 탭1~6에서 사이드바의 설정 요약이 사라진다. 정보가 필요한 경우 각 탭 내 UI에서 확인한다.

### B.2 사이드바 레이아웃

```
┌─────────────────────────────────┐
│        Smart QC Platform        │
│  [🔬 모델 탐색 플랫폼]    (버튼) │
│  [🏭 비전검사 플랫폼]     (버튼) │
└─────────────────────────────────┘
```

### B.3 버튼 스타일 규칙

| 상태 | `type` 파라미터 | 시각적 효과 |
|------|-----------------|-------------|
| 현재 활성 대시보드 | `"primary"` | 파란색 배경 (Streamlit 기본 primary 색상) |
| 비활성 대시보드 | `"secondary"` | 회색 아웃라인 |

```python
# components/sidebar.py 구현 기준 (04_System_Architecture.md B.10.1 참조)
st.sidebar.title("Smart QC Platform")
st.sidebar.markdown("---")
st.sidebar.button(
    "🔬 모델 탐색 플랫폼",
    use_container_width=True,
    type="primary" if active == "explorer" else "secondary"
)
st.sidebar.button(
    "🏭 비전검사 플랫폼",
    use_container_width=True,
    type="primary" if active == "inspection" else "secondary"
)
```

## D. 검사 탭1 — 실시간 검사 UI

### D.1 전체 레이아웃

```
┌──────────────────────────────────────────────────────────────────────┐
│  [🔍 수동 검사 (1개 검사)]  [▶ 자동 검사 (3초마다 1개)]  [⏹ 자동 검사 중지]  │
├──────────────┬──────────────────────┬──────────────────────┤
│  판정 결과    │     원본 이미지       │     Anomaly Map       │
│              │                      │                      │
│ ✅ 양품      │   [이미지 표시]       │  [히트맵 표시]        │
│ Score: 0.23  │                      │                      │
│              │   filename.png       │   Anomaly Map        │
├──────────────┴──────────────────────┴──────────────────────┤
│  (초기 상태: "검사 버튼을 눌러 시작하세요." 안내)           │
└──────────────────────────────────────────────────────────────────────┘
```

### D.2 버튼 영역

| 버튼 | 타입 | 활성화 조건 |
|------|------|-------------|
| `🔍 수동 검사 (1개 검사)` | `type="primary"` | `insp_auto_active == False` |
| `▶ 자동 검사 (3초마다 1개)` | `type="secondary"` | `insp_auto_active == False` |
| `⏹ 자동 검사 중지` | `type="secondary"` | `insp_auto_active == True` |

- 세 버튼은 `st.columns(3)` 또는 단일 행에 나란히 배치한다.
- 자동 검사 진행 중에는 수동 검사 버튼이 `disabled=True`가 된다.

### D.3 3열 결과 패널 (col1 / col2 / col3)

#### col1 — 판정 결과

| 상태 | 렌더링 |
|------|--------|
| 양품 | `st.success("✅ 양품")` + `st.metric("Anomaly Score", f"{score:.4f}")` |
| 불량 | `st.error("❌ 불량")` + `st.metric("Anomaly Score", f"{score:.4f}")` |
| 검사 전 | `st.info("검사 버튼을 눌러 시작하세요.")` |

#### col2 — 원본 이미지

```python
st.image(image_path, caption=image_name, use_container_width=True)
```

- 이미지가 없는 초기 상태에는 빈 영역 또는 안내 텍스트 표시.

#### col3 — Anomaly Map

```python
st.image(heatmap_rgb, caption="Anomaly Map", use_container_width=True)
```

- Anomaly Map은 jet colormap으로 변환 후 RGB numpy array로 전달한다.
- `matplotlib.cm.jet` 또는 `cv2.applyColorMap(map, cv2.COLORMAP_JET)` 사용.

### D.4 불량 감지 팝업 UI

팝업은 3열 패널 위에 `st.error()` 형태로 표시한다.

```
┌────────────────────────────────────────────────┐
│ ❌ 불량이 감지되었습니다! 자동 검사가 중지되었습니다. │
│                                                │
│  [✅ 확인 및 재개]      [🛑 검사 종료]           │
└────────────────────────────────────────────────┘
```

- `insp_defect_popup == True`인 경우 팝업만 렌더링하고 `return`으로 나머지 UI를 숨긴다.
- 팝업이 열린 동안 자동 검사 루프를 실행하지 않는다.

## E. 검사 탭2 — 검사 이력 및 통계 UI

### E.1 전체 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  검사 이력                              [📥 CSV 내보내기]     │
├────┬──────────────────┬──────────────┬────────────┬──────────┤
│ 번호│       시각        │   이미지명    │  판정결과  │  Score   │
├────┼──────────────────┼──────────────┼────────────┼──────────┤
│  5 │ 2026-05-26 14:32 │ crack_01.png │ 🔴 불량    │  0.8721  │
│  4 │ 2026-05-26 14:31 │ good_03.png  │ 🟢 양품    │  0.1243  │
│ .. │       ...        │     ...      │    ...     │   ...    │
├──────────────────────────────────────────────────────────────┤
│     총 검사    │    양품     │    불량     │   불량률   │
│       5        │      4      │      1      │   20.0%   │
└──────────────────────────────────────────────────────────────┘
```

### E.2 이력 테이블 스펙

| 컬럼 | 데이터 소스 | 표시 형식 |
|------|------------|----------|
| 번호 | `record.seq` | 정수 |
| 시각 | `record.inspected_at` | `YYYY-MM-DD HH:MM:SS` |
| 이미지명 | `record.image_name` | 문자열 |
| 판정결과 | `record.verdict` | "양품" (초록) / "불량" (빨간) |
| Anomaly Score | `record.anomaly_score` | `f"{score:.4f}"` |

- 정렬: `seq` 내림차순 (최신 검사 상단)
- 판정결과 컬러링: `df.style.apply()` 또는 pandas Styler 사용

### E.3 KPI 카드 스펙

```python
col1, col2, col3, col4 = st.columns(4)
col1.metric("총 검사", total)
col2.metric("양품", good_count)
col3.metric("불량", defect_count)
col4.metric("불량률", f"{defect_rate:.1%}")
```

- KPI 카드는 이력 테이블 **하단**에 배치한다.
- 검사 기록이 없을 때도 KPI 카드는 0 값으로 표시한다 (Guard는 이력 테이블에만 적용).

### E.4 CSV 내보내기 버튼

- 테이블 우측 상단에 배치 (`st.columns([3, 1])` 또는 `st.container()` 활용).
- `len(insp_records) == 0` 시 `disabled=True`.

### E.5 실시간 통계 차트 3분할 — v1.2 추가

기존 KPI 카드 아래에 3분할 레이아웃 추가.

```
┌────────────────────────────────────────────────────────────────────────┐
│  [단위 선택] [20개 단위]  [40개 단위]  [100개 단위]                       │
├─────────────────────┬────────────────────────┬─────────────────────────┤
│  [좌] 시간 범위 테이블│  [중] Anomaly Score      │  [우] Anomaly Score       │
│                     │      히스토그램           │      산점도               │
│  2026-06-24         │   y                      │  y  1.0                  │
│  14:00~14:01  ← 선택│   ▌                      │  ·  0.8  - - threshold   │
│  14:01~14:02        │ 10▌  ▌                   │  ●  0.6                  │
│  14:02~14:03 (진행) │   ▌  ▌  ▌                │  ○  0.4  ●               │
│                     │  0────────── x           │     0.2  ○─●             │
│                     │   0    0.5   1.0          │  ●  0.0                  │
│                     │  ■ 정상  ■ 불량            │     1  5  10 15 20   x   │
└─────────────────────┴────────────────────────┴─────────────────────────┘
```

**차트 상세 규칙:**

| 항목 | 히스토그램 (중앙) | 산점도 (우측) |
|------|-----------------|-------------|
| x축 | 0~1 고정 (Score) | 1~N 고정 (N=단위 크기) |
| y축 | 동적 (기본 0~10) | 0~1 고정 |
| 파란색 | 정상 판정 bar | 정상 판정 점 |
| 빨간색 | 불량 판정 bar | 불량 판정 점 |
| threshold 표시 | 수직 점선 | 수평 빨간 점선 |
| 연결선 | 없음 | 앞뒤 점 연결 |
| 차트 제목 | `Anomaly Score 분포 — {시간범위}` | `Anomaly Score 추이 — {시간범위}` |

**그룹 선택 상호작용:**
- 좌측 테이블 행 클릭 → 중앙·우측 차트 해당 그룹 데이터로 갱신
- 기본 선택: 가장 최근 그룹 (미완성 포함)
- 미완성 그룹: 탭 전환 시마다 최신 데이터 반영

| 컬럼 | 데이터 소스 | 표시 형식 |
|------|------------|----------|
| 실험명 | `experiment.experiment_id` | 문자열 (현재 모델은 "✅ 현재 적용 중" 접미사 추가) |
| 모델 타입 | `experiment.model_type` | "EfficientAD" / "PatchCore" |
| F1 Score | `experiment.metrics.f1_score` | `f"{f1:.4f}"` |
| AUC | `experiment.metrics.auc` | `f"{auc:.4f}"` |
| 실행 시각 | `experiment.created_at` | `YYYY-MM-DD HH:MM` |

- `status == "completed"` 실험만 표시.
- 기본 정렬: F1 Score 내림차순.
- 선택: `selection_mode="single-row"`, `on_select="rerun"`.

### F.3 모델 적용 버튼

| 항목 | 내용 |
|------|------|
| 레이블 | `"✅ 이 모델로 검사 시작"` |
| 타입 | `type="primary"` |
| 비활성화 조건 | 테이블에서 행이 선택되지 않은 경우 |
| 위치 | 테이블 하단 중앙 또는 우측 |

### F.4 경고 메시지

```python
st.warning("⚠️ 모델을 교체하면 현재 세션의 모든 검사 이력이 삭제됩니다.")
```

- 버튼 클릭 시 별도 확인 팝업 없이 즉시 교체한다 (단, 경고 메시지는 항상 표시).
- 이미 이력이 없는 경우(`insp_records == []`)에도 경고 메시지를 표시한다.

## G. 공통 UI 규칙

### G.1 Guard 메시지 스타일

| Guard 유형 | Streamlit 컴포넌트 | 예시 |
|------------|-------------------|------|
| 모델 미선택 | `st.info(...)` | "검사에 사용할 모델이 선택되지 않았습니다." |
| 이력 없음 | `st.info(...)` | "아직 검사 기록이 없습니다." |
| 완료 실험 없음 | `st.info(...)` | "사용 가능한 완료된 실험이 없습니다." |
| 불량 감지 팝업 | `st.error(...)` | "불량이 감지되었습니다!" |

### G.2 색상 규칙

| 의미 | Streamlit 표현 | 색상 |
|------|---------------|------|
| 양품 판정 | `st.success()` 또는 green text | 초록 |
| 불량 판정 | `st.error()` 또는 red text | 빨간 |
| 정보/안내 | `st.info()` | 파란 |
| 경고 | `st.warning()` | 노란 |

### G.3 반응형 레이아웃

- `use_container_width=True`를 이미지·버튼·데이터프레임에 기본 적용한다.
- `st.columns([a, b, c])` 비율은 내용에 따라 조정하되, 3열 검사 결과는 `[1, 2, 2]` 비율을 기본값으로 한다 (판정결과가 이미지보다 작음).

### G.4 초기 상태 처리

| 상황 | 표시 내용 |
|------|----------|
| 앱 첫 진입, 모델 미선택 | 탭1·탭2: Guard 메시지. 탭3: 실험 목록 표시 |
| 모델 선택 직후, 검사 전 | 탭1: 버튼 활성화 + "검사 버튼을 눌러 시작하세요." 안내 |
| 첫 검사 완료 | 탭1: 3열 결과 표시. 탭2: 이력 1개 + KPI 갱신 |

### G.5 자동 검사 상태 표시

- `insp_auto_active == True` 인 경우 탭1 상단에 `st.warning("🔄 자동 검사 진행 중...")` 배너를 표시한다.
- 배너는 버튼 영역 위에 배치한다.
